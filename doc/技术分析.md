# MediaToolkit 技术分析

## 媒体信息模块

### 概述

媒体信息模块（`src/media_info/`）是 MediaToolkit 的第一个功能模块，负责解析本地媒体文件的容器与编码信息，并在 GUI 中展示结果。模块采用 Qt 的线程模型将 I/O 密集型的 FFmpeg 调用隔离到独立线程，保证主线程（UI）始终响应。

---

### 文件结构

```
src/media_info/
├── MediaInfoWorker.h     # 数据结构 MediaInfo + Worker 类声明
├── MediaInfoWorker.cpp   # FFmpeg 解析逻辑
├── MediaInfoWidget.h     # UI 组件声明
└── MediaInfoWidget.cpp   # UI 布局、线程管理、结果展示
```

---

### 数据结构

```cpp
struct MediaInfo {
    QString filename;
    qint64  durationMs;   // 时长（毫秒）
    qint64  bitrate;      // 容器级比特率（bps）
    int     width;        // 视频宽度（像素）
    int     height;       // 视频高度（像素）
    QString videoCodec;   // 视频编码名称（如 h264、hevc）
    QString audioCodec;   // 音频编码名称（如 aac、mp3）
    int     sampleRate;   // 音频采样率（Hz）
    int     channels;     // 音频声道数
};
```

`MediaInfo` 是纯数据结构，通过 `Q_DECLARE_METATYPE` 注册，使其可在 Qt 信号槽中跨线程传递。

---

### 解析逻辑（MediaInfoWorker）

`MediaInfoWorker` 继承自 `QObject`，运行在独立 `QThread` 中。核心方法为 `doWork(const QString &filePath)`，流程如下：

**1. 打开容器**

```cpp
AVFormatContext *fmt = nullptr;
avformat_open_input(&fmt, filePath.toUtf8().constData(), nullptr, nullptr);
```

使用 `libavformat` 打开文件，支持 FFmpeg 能识别的所有容器格式（mp4、mkv、avi、mov、mp3、wav、flac 等）。

**2. 读取流信息**

```cpp
avformat_find_stream_info(fmt, nullptr);
```

部分格式（如 MPEG-TS）不在容器头部存储完整编码参数，此步骤通过解码少量数据包来探测流参数。

**3. 提取容器级信息**

```cpp
info.durationMs = (fmt->duration != AV_NOPTS_VALUE) ? fmt->duration / 1000 : 0;
info.bitrate    = fmt->bit_rate;
```

`fmt->duration` 的单位为 `AV_TIME_BASE`（微秒），除以 1000 转换为毫秒。

**4. 遍历流，提取编码参数**

```cpp
for (unsigned i = 0; i < fmt->nb_streams; ++i) {
    AVCodecParameters *par = fmt->streams[i]->codecpar;
    const AVCodecDescriptor *desc = avcodec_descriptor_get(par->codec_id);
    // 取第一条视频流 / 第一条音频流
}
```

仅取各类型的第一条流，忽略多路视频或多路音频的后续轨道。音频声道数通过 `par->ch_layout.nb_channels` 读取（FFmpeg 5.x 新 API）。

**5. 资源释放与信号发射**

```cpp
avformat_close_input(&fmt);
emit resultReady(info);
```

错误路径同样通过 `emit errorOccurred(message)` 通知 UI，不抛出异常。

### 已知限制

- 时长显示为原始毫秒数，未格式化为 `HH:MM:SS`。
- 仅提取第一条视频流和第一条音频流，忽略多轨道文件中的其余轨道。
- 比特率为容器级总比特率，非单独音/视频流的比特率。
- 不支持网络 URL（仅限本地文件路径）。

## 视频截图模块

### 概述

视频截图模块（`src/screenshot/`）允许用户输入时间戳后在视频中精确定位并截取单帧，保存为 PNG 文件，同时在界面内预览。模块分为两个异步操作：**探测**（加载文件元数据）和**截图**（seek + 解码 + 转换），均在独立线程中执行。

---

### 文件结构

```
src/screenshot/
├── ScreenshotWorker.h     # Worker 类声明（probeVideo + doWork）
├── ScreenshotWorker.cpp   # FFmpeg 探测与 seek 解码逻辑
├── ScreenshotWidget.h     # UI 组件声明
└── ScreenshotWidget.cpp   # UI 布局、线程管理、结果展示
```

---

### 解析逻辑（ScreenshotWorker）

`ScreenshotWorker` 继承自 `QObject`，运行在独立 `QThread` 中，提供两个槽方法。

#### probeVideo — 元数据探测

```cpp
void ScreenshotWorker::probeVideo(const QString &filePath)
```

流程：打开容器 → `avformat_find_stream_info` → 找最佳视频流 → 读取元数据 → 关闭容器 → emit。

关键字段提取：

```cpp
AVStream *st = fmt->streams[videoStream];
double fps   = av_q2d(st->avg_frame_rate);   // 帧率
int width    = st->codecpar->width;
int height   = st->codecpar->height;

// 总帧数：优先使用容器记录值，否则按 fps × 时长估算
qint64 totalFrames = st->nb_frames > 0
    ? st->nb_frames
    : (qint64)(fps * fmt->duration / AV_TIME_BASE);

double durationSec = (double)fmt->duration / AV_TIME_BASE;
```

失败时 emit `videoInfoReady(0, 0, 0, 0, 0.0)`，Widget 侧检测 `durationSec <= 0` 来区分成功与失败。

#### doWork — 时间戳 seek + 解码

```cpp
void ScreenshotWorker::doWork(const QString &filePath, double seconds)
```

**1. seek 到目标位置**

```cpp
int64_t seekTs = (int64_t)(seconds * AV_TIME_BASE);
av_seek_frame(fmt, -1, seekTs, AVSEEK_FLAG_BACKWARD);
avcodec_flush_buffers(ctx);   // 清空解码器内部缓冲
```

`AVSEEK_FLAG_BACKWARD` 保证 seek 到不晚于目标时间的最近关键帧（I-frame）。seek 失败时回退到文件开头，避免因容器索引缺失导致整体失败。`avcodec_flush_buffers` 是必要步骤——若不清空，解码器可能输出 seek 前残留的帧数据。

**2. 取第一帧**

```cpp
while (av_read_frame(fmt, pkt) >= 0 && !found) {
    if (pkt->stream_index == videoStream) {
        if (avcodec_send_packet(ctx, pkt) == 0) {
            while (avcodec_receive_frame(ctx, frame) == 0 && !found) {
                // 像素格式转换 → QImage → emit
                found = true;
            }
        }
    }
    av_packet_unref(pkt);
}
```

seek 后取**第一个可解码帧**而非精确匹配时间戳，这与主流播放器行为一致，且避免了因 B-frame reorder 导致的时间戳对比歧义。

**3. 像素格式转换**

```cpp
SwsContext *sws = sws_getContext(
    ctx->width, ctx->height, ctx->pix_fmt,       // 源：解码器原始格式（通常 YUV420P）
    ctx->width, ctx->height, AV_PIX_FMT_RGB24,   // 目标：Qt 兼容的 RGB24
    SWS_BILINEAR, nullptr, nullptr, nullptr);

QImage img(ctx->width, ctx->height, QImage::Format_RGB888);
uint8_t *dst[1]  = { img.bits() };
int dstStride[1] = { img.bytesPerLine() };
sws_scale(sws, frame->data, frame->linesize, 0, ctx->height, dst, dstStride);
sws_freeContext(sws);
```

`QImage::Format_RGB888` 与 `AV_PIX_FMT_RGB24` 内存布局完全一致（均为无 Alpha 的 R8G8B8 紧密排列），可直接将 `img.bits()` 作为 `sws_scale` 的输出缓冲，无需额外拷贝。

## 音频提取转换模块

### 概述

音频提取转换模块（`src/audio/`）从视频或音频文件中提取音轨，重采样后编码输出为 MP3 或 WAV，并通过进度信号实时更新 UI。相较于前两个模块，本模块引入了三个新的核心概念：**音频编码器**（`AVCodecContext` 写入方向）、**重采样器**（`SwrContext`）和**音频 FIFO 缓冲**（`AVAudioFifo`）。

---

### 文件结构

```
src/audio/
├── AudioWorker.h     # Worker 类声明（doExtract + cancel）
├── AudioWorker.cpp   # FFmpeg 解码-重采样-编码流水线
├── AudioWidget.h     # UI 组件声明
└── AudioWidget.cpp   # UI 布局、线程管理、进度展示
```

---

### 核心流水线（AudioWorker）

`AudioWorker` 继承自 `QObject`，运行在独立 `QThread` 中。核心方法为 `doExtract(inputPath, outputPath, format)`，整体流水线分为 12 个步骤：

```
avformat_open_input          打开输入文件
    │
avformat_find_stream_info    探测流参数
    │
av_find_best_stream          定位最佳音频流
    │
avcodec_open2 (decode)       打开解码器
    │
avformat_alloc_output_context2   创建输出格式上下文
    │
avcodec_open2 (encode)       打开编码器
    │
avformat_write_header        写容器头部
    │
swr_alloc_set_opts2 / swr_init   初始化重采样器
    │
av_audio_fifo_alloc          创建音频 FIFO
    │
循环：av_read_frame → avcodec_send/receive_frame
      → swr_convert → av_audio_fifo_write
      → [fifo≥frameSize] → avcodec_send/receive_frame → av_interleaved_write_frame
    │
刷新：解码器尾帧 → swr 残留 → FIFO 剩余 → 编码器 flush
    │
av_write_trailer             写容器尾部
```

#### 1. 输入侧（解封装 + 解码）

与 Phase 2/3 一致，使用 `avformat_open_input` + `avformat_find_stream_info` + `av_find_best_stream` 定位音频流，再通过 `avcodec_parameters_to_context` + `avcodec_open2` 打开解码器。

部分解码器（如某些 AAC 流）在 `avcodec_parameters_to_context` 后 `ch_layout.nb_channels` 为 0，需回退到默认立体声：

```cpp
if (decCtx->ch_layout.nb_channels == 0)
    av_channel_layout_default(&decCtx->ch_layout, 2);
```

#### 2. 输出侧（编码器 + 复用器）

**编码器选择**

| 输出格式 | FFmpeg Codec ID | 要求的采样格式 |
|----------|----------------|--------------|
| MP3 | `AV_CODEC_ID_MP3`（libmp3lame） | `AV_SAMPLE_FMT_FLTP`（平面浮点） |
| WAV | `AV_CODEC_ID_PCM_S16LE` | `AV_SAMPLE_FMT_S16`（交错 16-bit） |

编码器支持的采样格式通过 `encoder->sample_fmts[0]` 读取，避免硬编码：

```cpp
encCtx->sample_fmt = encoder->sample_fmts ? encoder->sample_fmts[0] : AV_SAMPLE_FMT_S16;
```

**输出格式上下文**

```cpp
avformat_alloc_output_context2(&outFmt, nullptr, "mp3", outputPath);
```

第三个参数为格式名称字符串（`"mp3"` / `"wav"`），FFmpeg 据此自动选择复用器（muxer）。对于需要全局头部的格式（`AVFMT_GLOBALHEADER`），须在 `avcodec_open2` 前设置 `AV_CODEC_FLAG_GLOBAL_HEADER`。

#### 3. 重采样器（SwrContext）

解码输出与编码器输入在三个维度上可能不匹配：**采样格式**、**采样率**、**声道布局**。`SwrContext` 统一处理这三种转换。

本模块使用 FFmpeg 7.x 新 API `swr_alloc_set_opts2`，接受 `AVChannelLayout*` 而非已废弃的 `uint64_t channel_layout`：

```cpp
swr_alloc_set_opts2(&swr,
    &encCtx->ch_layout, encCtx->sample_fmt, encCtx->sample_rate,  // 输出参数
    &decCtx->ch_layout, decCtx->sample_fmt, decCtx->sample_rate,  // 输入参数
    0, nullptr);
swr_init(swr);
```

重采样时，输出采样数需根据重采样器内部延迟（`swr_get_delay`）动态计算，避免丢弃尾部样本：

```cpp
int outSamples = (int)av_rescale_rnd(
    swr_get_delay(swr, decCtx->sample_rate) + decFrame->nb_samples,
    encCtx->sample_rate, decCtx->sample_rate, AV_ROUND_UP);
```

重采样结果写入临时 `AVFrame`，再推入 FIFO：

```cpp
AVFrame *swrFrame = av_frame_alloc();
// 设置 format / ch_layout / sample_rate / nb_samples
av_frame_get_buffer(swrFrame, 0);
int converted = swr_convert(swr, swrFrame->data, outSamples,
    (const uint8_t**)decFrame->data, decFrame->nb_samples);
av_audio_fifo_write(fifo, (void**)swrFrame->data, converted);
av_frame_free(&swrFrame);
```

#### 4. 音频 FIFO（AVAudioFifo）

libmp3lame 每帧固定消费 **1152 个采样**（`encCtx->frame_size = 1152`）。解码帧大小因解码器而异（AAC 通常 1024，MP3 1152，PCM 可变），重采样后同样不对齐。`AVAudioFifo` 充当弹性缓冲区，使编码器始终能取到整帧数据：

```
swr_convert → [AVAudioFifo] → 每次取 frameSize 个采样 → 编码器
```

对于 WAV（`AV_CODEC_CAP_VARIABLE_FRAME_SIZE`，`frame_size = 0`），`frameSize` 回退为 4096，同样走 FIFO 路径，保持代码统一。

```cpp
int frameSize = encCtx->frame_size > 0 ? encCtx->frame_size : 4096;
AVAudioFifo *fifo = av_audio_fifo_alloc(encCtx->sample_fmt,
    encCtx->ch_layout.nb_channels, frameSize * 4);
```

编码循环从 FIFO 取满帧，设置 PTS 后送入编码器：

```cpp
encFrame->pts = pts;   // pts 单位：采样数，time_base = {1, sample_rate}
pts += toRead;
avcodec_send_frame(encCtx, encFrame);
av_frame_unref(encFrame);  // 释放我们持有的引用，编码器内部仍持有副本

while (avcodec_receive_packet(encCtx, outPkt) == 0) {
    av_packet_rescale_ts(outPkt, {1, encCtx->sample_rate}, outStream->time_base);
    av_interleaved_write_frame(outFmt, outPkt);
}
```

#### 5. 三阶段刷新

编码结束时必须按顺序完成三次刷新，否则尾部数据丢失：

| 阶段 | 操作 | 原因 |
|------|------|------|
| ① 刷解码器 | `avcodec_send_packet(decCtx, nullptr)` | 取出解码器内部缓存的最后几帧 |
| ② 刷重采样器 | `swr_convert(swr, dst, n, nullptr, 0)` | 取出 swr 内部因延迟积累的残留采样 |
| ③ 刷编码器 | `avcodec_send_frame(encCtx, nullptr)` | 取出编码器内部已编码但尚未输出的包 |

刷新后调用 `av_write_trailer` 写容器尾部——对 WAV 尤为关键，因为 WAV 头部的数据块大小字段在此步骤才被回填。

#### 6. 取消机制

`m_cancel` 为 `std::atomic<bool>`，Worker 在 `av_read_frame` 主循环每次迭代检测。取消时跳过刷新阶段，直接进入资源清理，不写 trailer（输出文件不完整，由用户决定是否删除）。

#### 开始 / 取消切换

`m_running` 标志控制按钮语义：

```
m_running = false → 按钮文字 "开始提取"，点击后 emit requestExtract(...)，setRunning(true)
m_running = true  → 按钮文字 "取消"，点击后 emit requestCancel()，禁用按钮显示 "正在取消..."
```

Worker 完成（`finished` 或 `errorOccurred`）时调用 `setRunning(false)` 恢复所有控件。

#### 线程管理

```
AudioWidget（主线程）
    │
    ├── new AudioWorker
    ├── new QThread
    ├── worker->moveToThread(thread)
    └── thread->start()

信号连接（跨线程，自动 QueuedConnection）：
    AudioWidget::requestExtract     → AudioWorker::doExtract
    AudioWidget::requestCancel      → AudioWorker::cancel
    AudioWorker::progressUpdated    → AudioWidget::onProgress
    AudioWorker::finished           → AudioWidget::onFinished
    AudioWorker::errorOccurred      → AudioWidget::onError
    QThread::finished               → AudioWorker::deleteLater
```

### 已知限制

- 取消后输出文件残留磁盘，头部大小字段未回填（WAV 不可播放，MP3 可能可播放截断内容），需用户手动删除。
- MP3 输出依赖 FFmpeg 构建时包含 `libmp3lame`；若构建未含该库，`avcodec_find_encoder(AV_CODEC_ID_MP3)` 返回 `nullptr`，Worker 发出错误信号。
- 进度计算依赖音频流的 `pts` 字段；部分容器（如某些 AVI）音频流 `pts` 不连续，进度条可能跳变。
- 仅提取第一条音频流，多音轨文件（如含多语言配音的 MKV）无法选轨。
- 不支持网络 URL，仅限本地文件路径。
